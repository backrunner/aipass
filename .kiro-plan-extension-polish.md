## 3. Stop detecting arbitrary sites as "Custom AI Provider"
Decision: only fire on **known sites** OR **strong token-page signals**.
File: `apps/extension/src/content/detector.ts`
- Root cause: `buildDraft` (line 54-72) returns a draft whenever `provider || endpoint || secret` is truthy, with a generic "Custom AI Provider" fallback title. `findEndpoint`/secret scan fire on unrelated pages.
- Tighten `buildDraft` to require a **recognized context**:
  - `provider` matched by domain (known providers in `providerDefinitions`), OR
  - `guessSelfHostedProvider` matched a known gateway by brand hint (new-api/one-api/litellm/sub2api — already implemented, line 107-126), OR
  - a **strong token-page signal**: `SELF_HOSTED_TOKEN_PATH_PATTERN.test(pathname)` or `hasSelfHostedKeyPageText(doc)` AND a secret was actually captured.
  - If none of these hold → return `null` (no draft), instead of emitting a "Custom AI Provider" draft.
- Keep the `"Custom AI Provider"` title only for the recognized-but-unnamed self-hosted/endpoint case (so a real gateway without a brand name still works); remove the path where a bare endpoint/secret on a random page produces a draft.
- `detectAllFromDocument` (line 43-52): drop the `!candidates.length` fallback that builds a draft with no secret — only emit drafts that have a captured `apiKey` on a recognized page.
- Net effect: random sites with an `<input value="https://...">` no longer pop a "自定义提供商" draft; only known providers + recognized gateway/token pages do.
